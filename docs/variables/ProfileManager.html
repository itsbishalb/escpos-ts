    Registry for looking up and caching PrinterProfile instances.

    Profiles are sourced from the bundled capabilities.json database. Resolved profiles are cached in memory — repeated calls with the same name return the identical object reference.

    Type Declaration

    • getProfile: function
      • getProfile(name?: string): PrinterProfile

        Retrieve a PrinterProfile by name.

        Profiles are cached after the first load — subsequent calls with the same name return the cached instance without re-reading the database.

        Parameters

        • name: string = 'default'

          Profile name as it appears in capabilities.json (e.g. "default", "TM-T88V"). Defaults to "default".

        Returns PrinterProfile

        The resolved PrinterProfile instance.

        Throws

        Error if the named profile does not exist in the database.

    Example

    import { ProfileManager } from 'escpos-ts';

    const profiles = ProfileManager.listProfiles();
    // ['default', 'TM-T88V', 'POS-58', ...]

    const profile = ProfileManager.getProfile('TM-T88V');
    console.log(profile.getColumns('a')); // 42
